EXTENSION API

Vox Local Story TTS

Tài liệu API local để Chrome extension, automation script hoặc tool khác kết nối với hệ TTS Cô Gái Hoạt Ngôn.

Web UI Swagger OpenAPI JSON
Base URL mặc định: http://127.0.0.1:8010 Nếu `.env` bật LOCAL_API_KEY, LOCAL_API_KEYS hoặc LOCAL_API_KEY_HASHES, mọi endpoint có nhãn Auth cần header X-API-Key.

Flow khuyên dùng cho extension

  1. Gọi GET /health để kiểm tra server và lấy max_text_length.
  2. Gọi GET /v1/voices để chọn voice, ưu tiên Cô Gái Hoạt Ngôn.
  3. Gọi POST /v1/text/prepare để chia block, bỏ emotion tag và tính rate.
  4. Dùng POST /v1/tts cho một block hoặc POST /v1/tts/batch cho nhiều block.
  5. Tải lại audio bằng GET /v1/cache/{cache_id}?filename=....

Endpoint tự mô tả

Extension nên đọc endpoint này thay vì hard-code giới hạn hoặc danh sách emotion tag. 

GET /v1/extension/capabilities

Auth

Khi server bật LOCAL_API_KEY, LOCAL_API_KEYS hoặc LOCAL_API_KEY_HASHES, extension cần gửi header này cho các endpoint Auth. /health, /v1/voices và /v1/extension/capabilities vẫn mở để extension có thể tự khám phá server. 

fetch("http://127.0.0.1:8010/v1/providers", {
  headers: { "X-API-Key": "<LOCAL_API_KEY>" }
})

Web UI có thể bật session riêng bằng WEB_AUTH_ENABLED=true. Khi đó người dùng đăng nhập tại /login; API gọi từ chính trình duyệt sẽ dùng cookie session, còn extension/app ngoài vẫn nên dùng X-API-Key.

Google OAuth

Google OAuth là tùy chọn cho Web UI. Server yêu cầu GOOGLE_OAUTH_CLIENT_ID, GOOGLE_OAUTH_CLIENT_SECRET, GOOGLE_OAUTH_REDIRECT_URI và allowlist email/domain. Với LAN IP thuần, API key thường dễ vận hành hơn; Google OAuth phù hợp hơn khi dùng localhost hoặc domain HTTPS. 

WEB_AUTH_ENABLED=true
WEB_AUTH_PROVIDERS=api_key,google
GOOGLE_OAUTH_REDIRECT_URI=http://127.0.0.1:8010/auth/google/callback
GOOGLE_ALLOWED_EMAILS=alice@gmail.com,bob@gmail.com

Voice

Gọi /v1/voices để lấy các trường cần gửi vào /v1/tts. Extension nên lưu nguyên voice_type và resource_id. 

GET /v1/voices
[
  {
    "name": "Cô Gái Hoạt Ngôn",
    "voice_type": "BV074_streaming",
    "resource_id": "7102355709945188865",
    "language": "vi-VN",
    "provider": "capcut",
    "provider_label": "CapCut API qua K07VN/capcut-tts-api"
  }
]

Chuẩn bị text

Endpoint này không tạo audio. Nó giúp extension chia nội dung dài thành block hợp lệ, bỏ emotion tag khỏi text gửi TTS và tính rate/pause cho từng block. 

POST /v1/text/prepare
Content-Type: application/json
X-API-Key: <LOCAL_API_KEY nếu có>

{
  "text": "[buồn] Anh im lặng rất lâu...\n[cao trào] Nhưng mọi chuyện đổi khác.",
  "rate": 1.0,
  "chunk_size": 3200
}
{
  "total_chunks": 2,
  "total_chars": 58,
  "max_text_length": 3800,
  "chunk_size": 3200,
  "base_rate": 1.0,
  "chunks": [
    {
      "index": 1,
      "text": "Anh im lặng rất lâu...",
      "char_count": 22,
      "rate": 0.9,
      "emotion_tag": "[buồn]",
      "emotion_label": "Buồn",
      "pause_after": 0.55
    }
  ]
}

Tạo một audio

POST /v1/tts
Content-Type: application/json
X-API-Key: <LOCAL_API_KEY nếu có>

{
  "text": "Nội dung cần đọc",
  "rate": 1.0,
  "filename": "chuong-001.mp3",
  "voice_type": "BV074_streaming",
  "resource_id": "7102355709945188865",
  "voice_name": "Cô Gái Hoạt Ngôn"
}

Response là file audio binary. Các header quan trọng:

  • X-TTS-Cache: HIT hoặc MISS.
  • X-TTS-Cache-Id: ID để gọi /v1/cache/{id}.
  • X-TTS-Voice-Name: voice thực tế đã dùng, URL-encoded.
  • X-TTS-Fallback: true nếu provider chính lỗi và server fallback.

Tạo batch audio

Endpoint này phù hợp extension vì response là JSON, mỗi item có download_url trỏ về cache. Giới hạn mặc định là 100 block/lần gọi. 

POST /v1/tts/batch
Content-Type: application/json
X-API-Key: <LOCAL_API_KEY nếu có>

{
  "text": "Nội dung dài...",
  "rate": 1.0,
  "chunk_size": 3200,
  "filename_prefix": "truyen-dem-khuya",
  "voice_type": "BV074_streaming",
  "resource_id": "7102355709945188865",
  "voice_name": "Cô Gái Hoạt Ngôn"
}
{
  "total": 2,
  "items": [
    {
      "index": 1,
      "filename": "truyen-dem-khuya_part_001.mp3",
      "cache_id": "sha256...",
      "download_url": "/v1/cache/sha256...?filename=truyen-dem-khuya_part_001.mp3",
      "content_type": "audio/mpeg",
      "extension": "mp3",
      "cache": "MISS",
      "provider": "capcut",
      "provider_label": "CapCut API qua K07VN/capcut-tts-api",
      "voice": "Cô Gái Hoạt Ngôn",
      "voice_type": "BV074_streaming",
      "rate": 1.0,

      "emotion_label": "Bình thường"

  ]
}

Emotion tag

Các tag đặt ở đầu dòng/block. Server bỏ tag khỏi text audio và chỉ trả rate/pause đã tính.

[buồn] Anh im lặng rất lâu...
[cao trào] Nhưng điều anh nói sau đó khiến tất cả chết lặng.
[thì thầm] Và bí mật ấy... chưa từng thuộc về riêng anh.

Danh sách tag động:

GET /v1/emotion-tags

Manifest quyền cho Chrome extension

{
  "permissions": ["activeTab", "scripting", "storage", "downloads"],
  "host_permissions": [
    "http://127.0.0.1/*",
    "http://localhost/*"
  ]
}
